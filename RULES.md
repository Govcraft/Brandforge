# RULES.md - BrandForge Framework Actionable Rules

Simple actionable rules for BrandForge brand strategy framework operation.

## Core Operational Rules

### Task Management Rules
- TodoRead() → TodoWrite(3+ tasks) → Execute → Track progress
- Use batch tool calls when possible, sequential only when dependencies exist
- Always validate before execution, verify after completion
- Run brand quality gates before marking tasks complete
- Use /brandforge task and /brandforge spawn for complex multi-session workflows
- Maintain ≥90% context retention across brand development operations

### Brand Asset Security
- Always use Read tool before Write or Edit operations on brand assets
- Use absolute paths only, prevent path traversal attacks
- Prefer batch operations and transaction-like behavior for brand consistency
- Never commit brand changes automatically unless explicitly requested
- Validate brand guidelines compliance before any asset modifications

### Framework Compliance
- Check brand guidelines before any creative modifications
- Follow existing brand patterns and conventions
- Use project's established brand voice and visual identity
- Respect brand lifecycle phases and best practices
- Maintain consistency across all brand touchpoints

### Systematic Brand Changes
- **MANDATORY**: Complete brand ecosystem discovery before any changes
- Search ALL brand assets for ALL variations of target elements
- Document all brand references with context and impact assessment
- Plan update sequence based on brand dependencies and relationships
- Execute changes in coordinated manner following brand governance
- Verify completion with comprehensive post-change brand audit
- Validate related brand functionality remains consistent
- Use Task tool for comprehensive brand searches when scope uncertain

## Research Safety Protocols

### Synthetic User Research
- **Transparency Disclosure**: Always disclose synthetic nature to stakeholders
- **Validation Requirements**: Cross-validate with real user research
- **Bias Prevention**: Actively prevent perpetuation of harmful stereotypes
- **Appropriate Use**: Use synthetic research for concept validation, not final decisions
- **Ethical Review**: All synthetic research undergoes ethical review

### Traditional User Research
- **Consent Documentation**: Detailed consent forms for all participants
- **Privacy Protection**: Anonymization of all user data
- **Cultural Sensitivity**: Respectful engagement with diverse populations
- **Vulnerable Population Safeguards**: Extra protections for sensitive groups
- **Data Security**: Secure handling of all research data

### Market Research Compliance
- **Competitive Intelligence Ethics**: Fair and legal competitive analysis
- **Data Source Verification**: Validate all market data sources
- **Cultural Context Respect**: Sensitive handling of cultural market data
- **Regulatory Compliance**: Adherence to all relevant research regulations
- **Industry Standard Alignment**: Follow established market research practices
