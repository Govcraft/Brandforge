# /brandforge forge - Complete Brand Development Orchestration

## Command Structure
```yaml
---
command: "/brandforge forge"
category: "Brand Strategy Orchestration"
purpose: "End-to-end brand development with compound intelligence"
wave-enabled: true
performance-profile: "optimization"
---
```

## Purpose
Master orchestration command that coordinates all brand development phases through intelligent wave progression. Combines strategy, creative, research, and validation into unified brand development campaign.

## Usage Patterns
```bash
/brandforge forge [brand-name] [scope]
/brandforge forge --complete --enterprise
/brandforge forge --strategy --creative --launch
/brandforge forge --mvp --validate --iterate
```

## Arguments
- **brand-name**: Primary brand identifier
- **scope**: Development scope (mvp|complete|enterprise)

## Flags
- `--complete`: Full brand development lifecycle
- `--strategy`: Focus on strategic development
- `--creative`: Emphasize creative development
- `--research`: Include comprehensive research
- `--validate`: Continuous validation checkpoints
- `--launch`: Include launch orchestration
- `--mvp`: Minimum viable brand approach
- `--enterprise`: Enterprise-scale development
- `--iterate`: Iterative improvement cycles
- `--wave-mode [strategy]`: Wave orchestration strategy

## Wave Strategies
- **progressive**: Incremental brand enhancement
- **systematic**: Methodical development phases
- **adaptive**: Dynamic configuration based on feedback
- **enterprise**: Large-scale orchestration

## Auto-Persona Activation
- **architect**: Brand architecture and system design
- **frontend**: Visual identity and user experience
- **analyst**: Market intelligence and validation
- **mentor**: Strategic guidance and best practices
- **scribe**: Documentation and brand standards

## MCP Server Integration
- **Sequential**: Multi-phase orchestration and coordination
- **Context7**: Brand strategy patterns and frameworks
- **Magic**: Visual brand development and design systems
- **Playwright**: Brand testing and user validation

## Wave Architecture
### Wave 1: Foundation
- Market analysis and positioning
- Brand architecture definition
- Strategic framework establishment

### Wave 2: Creative Development
- Visual identity design
- Brand voice and messaging
- Creative asset development

### Wave 3: Validation
- Market testing and feedback
- Brand performance validation
- Iterative refinement

### Wave 4: Launch Preparation
- Launch strategy development
- Asset finalization
- Go-to-market orchestration

### Wave 5: Optimization
- Performance optimization
- Continuous improvement
- Scaling strategies

## Quality Gates
- **Wave Boundaries**: Validation at each wave completion
- **Cross-Wave Validation**: Integrated quality checks
- **Progressive Enhancement**: Each wave builds on previous
- **Rollback Capability**: Safe rollback on validation failures

## Examples
```bash
# Complete brand development
/brandforge forge "Nexus Innovations" --complete

# MVP approach with validation
/brandforge forge --mvp --validate --iterate

# Enterprise-scale with specific focus
/brandforge forge --strategy --creative --launch --enterprise

# Progressive enhancement
/brandforge forge --wave-mode progressive --iterate
```
