# /brandforge build - Asset Development

## Command Structure
```yaml
---
command: "/brandforge build"
category: "Brand Asset Development"
purpose: "Comprehensive brand asset creation and development"
wave-enabled: true
performance-profile: "optimization"
---
```

## Purpose
Systematic brand asset development that creates comprehensive brand toolkits, marketing materials, digital assets, and implementation guides. Builds production-ready brand assets with consistent quality and strategic alignment.

## Usage Patterns
```bash
/brandforge build [asset-type] --scope [level]
/brandforge build --toolkit --templates --guidelines
/brandforge build --digital --print --environmental
/brandforge build --scalable --consistent --quality
```

## Arguments
- **asset-type**: Specific asset category to build
- **scope**: Development scope and complexity level

## Asset Types
- **toolkit**: Complete brand toolkit development
- **templates**: Brand template creation
- **guidelines**: Brand guideline development
- **digital**: Digital asset creation
- **print**: Print material development
- **environmental**: Physical/environmental assets
- **merchandise**: Branded merchandise
- **presentations**: Branded presentation materials

## Flags
- `--toolkit`: Complete brand toolkit development
- `--templates`: Brand template and asset creation
- `--guidelines`: Brand guideline documentation
- `--digital`: Digital asset creation and optimization
- `--print`: Print material development
- `--scalable`: Scalable asset systems
- `--consistent`: Brand consistency enforcement
- `--quality`: Quality assurance and standards
- `--accessibility`: Accessibility compliance
- `--responsive`: Responsive design systems
- `--optimization`: Asset optimization and performance
- `--testing`: Asset testing and validation

## Auto-Persona Activation
- **frontend**: Asset user experience and visual design
- **qa**: Asset quality assurance and consistency
- **performance**: Asset optimization and efficiency
- **scribe**: Asset documentation and guidelines

## MCP Server Integration
- **Magic**: Visual asset creation and design systems
- **Context7**: Asset development frameworks and standards
- **Playwright**: Asset testing and user experience validation

## Asset Development Framework

### Brand Toolkit Development
- **Logo System**: Primary, secondary, and responsive logos
- **Color System**: Primary, secondary, and neutral color palettes
- **Typography System**: Font hierarchy and usage guidelines
- **Icon System**: Icon library and usage standards
- **Photography Style**: Image treatment and style guidelines
- **Layout Systems**: Grid systems and composition standards

### Digital Asset Creation
- **Website Assets**: Website graphics, banners, and elements
- **Social Media**: Platform-specific social media assets
- **Email Templates**: Branded email marketing templates
- **Digital Ads**: Online advertising creative assets
- **App Assets**: Mobile app interface elements
- **Video Assets**: Brand video templates and elements

### Print Material Development
- **Business Stationery**: Business cards, letterhead, envelopes
- **Marketing Materials**: Brochures, flyers, posters
- **Packaging**: Product packaging and labeling
- **Signage**: Environmental and directional signage
- **Presentations**: Branded presentation templates
- **Reports**: Branded document and report templates

### Template Systems
- **Design Templates**: Reusable design templates
- **Content Templates**: Content creation templates
- **Social Templates**: Social media content templates
- **Email Templates**: Email marketing templates
- **Presentation Templates**: Slide deck templates
- **Document Templates**: Business document templates

### Scalable Systems
- **Responsive Design**: Multi-device compatibility
- **Scalable Graphics**: Resolution-independent assets
- **Modular Systems**: Modular and flexible asset systems
- **Version Control**: Asset versioning and management
- **Asset Libraries**: Centralized asset storage and access
- **Usage Guidelines**: Asset usage and implementation guides

## Quality Gates
- **Brand Consistency**: Consistent brand application across all assets
- **Scalability**: Assets scale effectively across applications
- **Accessibility**: Assets meet accessibility standards
- **Performance**: Optimized for fast loading and performance
- **Quality Standards**: High-quality production standards
- **User Experience**: Positive user experience across all touchpoints

## Examples
```bash
# Complete brand toolkit development
/brandforge build --toolkit --templates --guidelines --scalable

# Digital asset creation
/brandforge build --digital --responsive --optimization --testing

# Print material development
/brandforge build --print --consistent --quality --accessibility

# Template system creation
/brandforge build --templates --scalable --consistent --guidelines

# Comprehensive asset development
/brandforge build --toolkit --digital --print --merchandise
```
